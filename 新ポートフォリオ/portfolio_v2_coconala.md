# Sunny Side Design — ポートフォリオサイト 設計書 v2.0
## ＜ ココナラ特化・Concept Work対応版 ＞

> **設計の大前提：** このポートフォリオの唯一のゴールは「ページを見た人が、ココナラのプロフィールを開いてメッセージを送ること」。すべての設計判断はこの一点から逆算する。

---

## 0. ロゴカラーと設計コンセプトの統合

ロゴを分析した結果、以下のカラー体系をポートフォリオ全体に統一します。

```
ロゴ Sun:      #F5C518 → #F5A623  (Yellow Gold → Warm Orange)
ロゴ Pencil:   #F5A623 → #E07A5F  (Warm Orange → Terracotta)   ← サイトのメインアクセント
ロゴ Text:     #3D3530             (Dark Charcoal)               ← メインテキスト
Background:    #F9F6F0             (Warm Off-White)
Sub Accent:    #81B29A             (Sage Green / 成果・信頼の表現)
```

**ロゴが体現するブランドメッセージ：**
太陽の「暖かさ・明るさ」と、ペンの「クラフト・精度」の融合。
これはDESIGN.mdの哲学「**Crafted, not Computed**」と完全に一致します。
ポートフォリオ全体を通じて「温かみある設計力」を一貫して演出してください。

## 1. サイトの全体構成（サイトマップ）

### 構成方針：**ワンページ ＋ 作品詳細のみ別ページ（ハイブリッド型）**

すべてのCTAは「ここから依頼 → ＜ cocona.la/... ＞」のみ。
フォーム・メアド・SNS DMへの誘導は完全排除。

---

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  MAIN PAGE（ワンページ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [ 00 ] FIXED HEADER
         ロゴ | Works | About | Skill | ─── [ ✦ ココナラで相談する ]

  [ 01 ] HERO  ★ 最重要セクション
         キャッチコピー / サブコピー / メインCTA（ボタン）
         ＋ 視覚的な作品サムネイルのチラ見せ

  [ 02 ] PHILOSOPHY（設計哲学・Why）
         「なぜデザインに心理学が必要か」を短く語る
         ─ 技術者としての差別化を早期に印象づける

  [ 03 ] CONCEPT WORKS（架空案件ギャラリー）
         2〜3件のコンセプトワークをカードで展示
         各カードクリック → 作品詳細ページへ

  [ 04 ] SKILL & TOOL（スキルセット）
         得意領域・使用ツール・対応範囲を視覚的に整理

  [ 05 ] FLOW（ご依頼の流れ）
         ━ ①ご相談（ コ コ ナ ラ ）→ ②ヒアリング → ③設計 → ④デザイン → ⑤納品
         ─ ステップ①に「ここから相談できます ↗」リンクを自然に配置

  [ 06 ] FAQ（よくある質問）
         「どんな案件を受けていますか？」
         「実績がない場合でも依頼できますか？」など、
         ─ ここもアンカーとしてコンバージョンを支える

  [ 07 ] CTA BANNER（最終誘導）
         ページ末尾に大きなバナー形式のCTA
         「まずはお気軽に。ご相談はこちらから。」

  [ FOOTER ]
         ロゴ / コピーライト / ─── [ ✦ ここから依頼する（ ▶ ）]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SUB PAGE（作品詳細） /works/[slug]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ※ 次章で詳述
```

> **💡 設計メモ：**
> 問い合わせフォームを排除した代わりに、CTAをFIXED HEADER・HERO・FLOW・CTA BANNERの**4箇所**に配置します。「気づいたときにすぐ押せる」状態を常に保つことが、外部フォームがなくてもコンバージョン率を維持する鍵です（Fittsの法則：常に「近い場所」にCTAを置く）。

## 2. Concept Work 詳細ページ テンプレート

### 基本方針：「結果の代わりに、思考を売る」

実績のないうちに「CVR〇〇%改善」と書けないのは事実です。
しかし、**設計の意図と仮説を言語化する深さ**は、実務経験者ですら多くがやっていません。
ここに圧倒的な差別化のチャンスがあります。

---

### ページ構成テンプレート（1作品あたり）

```
────────────────────────────────────────────────────────────
01. PROJECT HEADER
────────────────────────────────────────────────────────────
  [ タグ: Concept Work · 制作年月 · 担当: UX設計 / デザイン ]

  # 作品タイトル（例: 人材マッチングアカデミー「NextGate」LP）

  > 一行サマリー（例：「第二新卒〜キャリアチェンジ層の不安を
  >  『段階的な信頼構築』で解消するLP設計の仮説研究」）

  ━━ Concept Work について ━━━━━━━━━━━━━━━━━━━━━━
  本作品は実在する企業・サービスとは無関係の、
  デザイン設計力の実証を目的とした架空のコンセプト作品です。
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

────────────────────────────────────────────────────────────
02. HERO VISUAL（ファーストビジュアル）
────────────────────────────────────────────────────────────
  LP全体のヒーローセクション部分の大判モックアップ
  ※ PC + スマホの2画面モックアップが理想

────────────────────────────────────────────────────────────
03. THE BRIEF（架空の依頼背景）
────────────────────────────────────────────────────────────
  ┌─────────────────┬──────────────────────────────────────┐
  │ 想定クライアント  │ （例）転職・キャリア支援スクールのLP   │
  ├─────────────────┼──────────────────────────────────────┤
  │ 想定ターゲット    │ 25〜35歳・転職を考えているが迷っている層 │
  ├─────────────────┼──────────────────────────────────────┤
  │ 想定の課題       │ 競合が多く差別化が難しい。LP流入はある  │
  │                  │ がCVRが低い（想定: 0.8%前後）          │
  ├─────────────────┼──────────────────────────────────────┤
  │ 達成したいKPI    │ 無料相談予約のCVR: 0.8% → 2.0% 以上   │
  └─────────────────┴──────────────────────────────────────┘

────────────────────────────────────────────────────────────
04. HYPOTHESIS（課題仮説とアプローチ）
────────────────────────────────────────────────────────────

  ▼ 課題仮説
  「転職を検討中のユーザーが行動できない最大の理由は
   『能力不足への不安』と『失敗リスクへの恐れ』である。
   つまり、CVRを下げているのは意欲の低さではなく、
   ＜心理的障壁の高さ＞だ。」

  ▼ DRMアプローチ（ダイレクトレスポンスマーケティングの視点）
  ・ヒーローセクション: 「現状苦痛（pain）」に共感してから
    「理想の未来（gain）」を提示する PAINモデル を採用
  ・ソーシャルプルーフ: 数値よりも「類似ペルソナの声」を優先
    （「自分と同じ状況の人が成功した」という類似性の原理）
  ・CTAの設計: 「申し込む」ではなく「まず話してみる」という
    低コミットメント言語でクリックのハードルを下げる

  ▼ 使用した心理学的原則
  ┌──────────────────┬─────────────────────────────────────┐
  │ Fittsの法則        │ スマホでの親指操作を考慮し、主要CTA    │
  │                    │ を画面下部60%エリアに複数配置           │
  ├──────────────────┼─────────────────────────────────────┤
  │ ゲシュタルト近接   │ 「受講者の声」と「申込ボタン」を意図的  │
  │                    │ に隣接させ、信頼→行動の流れを短絡化    │
  ├──────────────────┼─────────────────────────────────────┤
  │ F字視線パターン    │ キャッチコピー（左上）→ サービス価値    │
  │                    │ （左寄せ）→ CTA（左寄り）で視線を誘導  │
  ├──────────────────┼─────────────────────────────────────┤
  │ 認知負荷の軽減     │ 料金・コース情報を3項目以内に絞り、     │
  │                    │ 「選択肢過多による離脱」を回避         │
  └──────────────────┴─────────────────────────────────────┘

────────────────────────────────────────────────────────────
05. FULL PAGE SCROLL（LP全体画像）
────────────────────────────────────────────────────────────
  縦長のフルスクロール画像でLPの全体像を提示

────────────────────────────────────────────────────────────
06. CLOSE-UP FOCUS（こだわりポイント × 2〜3箇所）
────────────────────────────────────────────────────────────
  【Focus 01】ヒーローセクション
  ─ 拡大画像 ＋ 「なぜこのレイアウトにしたか」の解説テキスト
  （例: 「左上に"転職で失敗したくない"という共感コピーを置き、
        直下にベネフィットを3点に絞って箇条書き。
        Fパターンの視線の起点に"不安への共感"を配置することで
        ユーザーの離脱を防ぐ仮説に基づいています。」）

  【Focus 02】CTAセクション
  ─ 拡大画像 ＋ 意図解説
  （例: 「ボタンテキストを"申し込む"から"まず話してみる（無料）"に
        変更。心理的コストを下げるマイクロコピーの設計事例です。」）

  【Focus 03】ソーシャルプルーフ
  ─ 拡大画像 ＋ 意図解説
  （例: 「星評価ではなく、ペルソナに近い年齢・職種を明示した顔写真付き
        コメントを採用。類似性の原理により共感度を高める設計です。」）

────────────────────────────────────────────────────────────
07. REFLECTION（設計を終えての考察）
────────────────────────────────────────────────────────────
  ▼ もし実際に公開・計測できたなら検証したいこと（仮説）
  ・ヒーローの画像を人物ありとなしでA/Bテスト
  ・CTA文言を「今すぐ」型と「まず〜」型で比較
  ・フォームのステップ数を1段階 vs 3段階で比較

────────────────────────────────────────────────────────────
08. NEXT WORK ／ CTA（ページ末尾）
────────────────────────────────────────────────────────────
  「このような設計でLPを作ってほしい方はこちら ▶」
  → ＜ cocona.la/... ＞ へのリンク（ターゲット：_blank）
```

## 3. コピーライティング

### 3-A. Heroセクション キャッチコピー（3パターン）

---

#### ▶ パターン 01 ｜「仮説と設計」で専門性を全面に出す型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  「なぜ離脱するのか」を
  先に設計する、LP制作を。

サブコピー:

  Sunny Side Designは、行動心理学とDRMの視点から
  「ユーザーが動かない本当の理由」を仮説として設定し、
  それを解消するワイヤーフレームとビジュアルを構築します。

  きれいなLPではなく、"機能するLP"を届けたい方へ。

CTA ボタン（後述）:
  ▶ ここから相談する（ ✦ ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**狙い:** 「デザイナー」ではなく「UXストラテジスト」としての価値を打ち出す。競合の多いデザイン市場で、思考の深さで差別化。マーケ責任者に刺さるキーワード（行動心理学・DRM）を前面に。

---

#### ▶ パターン 02 ｜「温かみ×結果」でブランドコンセプトを体現する型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  見た瞬間に、
  「ここは信頼できる」と感じるLPを。

サブコピー:

  完璧で冷たいAIデザインが溢れる時代に、
  あえて"人の手の温もり"を選ぶ。

  Sunny Side Designは、ユーザーの不安を取り除き
  「読んで、信じて、動く」導線を、ひとつずつ丁寧に設計します。

CTA ボタン:
  ▶ ことばを送ってみる（ ✦ ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**狙い:** ロゴの「Sunny Side（明るさ・温もり）」コンセプトと完全に一致。「AI時代の逆張り」という文脈でブランドの独自性を打ち出しつつ、結果（機能するLP）も訴求。

---

#### ▶ パターン 03 ｜「問いかけ」で離脱率を最小化する共感型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  そのLP、
  なぜ成約しないか——説明できますか？

サブコピー:

  「デザインをきれいにしたのに、結果が変わらない」
  その原因は、デザインの前段階にあることがほとんどです。

  Sunny Side Designは、ターゲットの心理と導線を先に設計し、
  「なぜここでクリックするのか」まで言語化した上でデザインします。

CTA ボタン:
  ▶ 設計力を試してみてください（ ✦ ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**狙い:** 「きれいなデザインを依頼したのに結果が出なかった」という経験を持つクライアントのペインに直接触れる。Concern（懸念）を引き出す問いかけ型で思わず読み進めさせる構造。

---

> ### ✅ 推奨パターン
> **ローンチ時はパターン 01、定着後にパターン 03 で A/Bテストを推奨。**
> パターン 01 は「何をやっている人か」が最も速く伝わり、ファーストインプレッションに強い。パターン 03 は読み進めた人の共感度が高く、コンバージョン率が高い可能性がある反面、誤解を与えるリスクもあるため実績が積まれた段階で投入する。

### 3-B. ココナラ誘導 CTA（パターン集）

ここが通常の制作依頼との最大の差分です。
**「依頼させる」のではなく「話しかけたくなる」文脈設計**が重要です。

---

#### ▶ メインCTA（ボタン + 誘導文セット）

**パターン A ｜ 敷居を下げる「まず話してみる」型**

```
── 誘導文 ──────────────────────────────────────────────
  「どんな些細な相談でも大丈夫です。
   価格・内容・スケジュール、まずはお気軽に。」

── ボタンテキスト ───────────────────────────────────────
  ╔══════════════════════════════════╗
  ║  ✦ ここからご相談・ご依頼へ      ║
  ╚══════════════════════════════════╝
  ↳ ※ ococona.laのプロフィールページを開きます

── ボタン直下の補足（マイクロコピー）───────────────────
  ブラウザの別タブでプロフィールが開きます。
  お問い合わせはページ内の「メッセージを送る」から。
```

---

**パターン B ｜ 価値提案型「無料相談」強調型**

```
── 誘導文 ──────────────────────────────────────────────
  「まずはあなたのLPの課題をお聞かせください。
   ヒアリングの中でお役に立てる提案をします。」

── ボタンテキスト ───────────────────────────────────────
  ╔══════════════════════════════════╗
  ║  ✦ 無料相談をはじめる（ ▶ ）     ║
  ╚══════════════════════════════════╝

── ボタン直下の補足 ──────────────────────────────────────
  ご依頼の確約は不要です。まずお気軽にどうぞ。
```

---

**パターン C ｜ 緊急性・希少性を自然に演出する型（FLOW内など）**

```
── 誘導文 ──────────────────────────────────────────────
  「1ヶ月の受注枠には限りがあります。
   お早めにご相談いただけると確実です。」

── ボタンテキスト ───────────────────────────────────────
  ╔══════════════════════════════════╗
  ║  ✦ 空き状況を確認する（ ▶ ）     ║
  ╚══════════════════════════════════╝
```

---

> **🔑 ボタン配置の原則（Fittsの法則 × ファネル設計）**
> | 配置場所 | 推奨パターン | 理由 |
> |---|---|---|
> | FIXED HEADER（右端） | ミニマム版「ご相談はこちら」 | 常に視界に入れる |
> | HERO（本文直下） | パターン A | 初回訪問者の温度感に合わせる |
> | FLOW セクション内 | パターン C | 依頼プロセスが見えた直後が最も前向き |
> | CTA BANNER（末尾） | パターン B | 最後まで読んだ高関与ユーザーへの最終誘導 |

## 4. 注意事項・規約対応チェックリスト

サイト公開前に以下を必ず確認してください。

- [ ] お問い合わせフォーム・メールアドレスが一切掲載されていない
- [ ] Twitter / Instagram / LINEへの誘導リンクが含まれていない
- [ ] 「ココナラ外でのお支払い」を示唆する表現がない
- [ ] 外部リンクはすべてcocona.laドメインのみ（target="_blank"推奨）
- [ ] Concept Workに「架空案件である旨」の但し書きが記載されている
- [ ] 実在する企業・ブランド名をモックアップに使用していない
