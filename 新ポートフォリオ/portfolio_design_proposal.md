# Sunny Side Design — ポートフォリオサイト 全体設計・構成案

> **Crafted, not Computed.** 計算で生み出されたのではなく、確かな体温を持つクリエイターとしての価値を証明するポートフォリオ。

---

## 1. デザインコンセプトと世界観の定義

### コアコンセプト：「Warm Precision（温もりある精度）」

AIが生成したような「完璧で冷たいデザイン」が溢れる現代において、Sunny Side Designのポートフォリオは **意図的な"揺らぎ"と、科学的根拠に基づくUX設計** の共存を体現します。

訪問者が受け取るべきメッセージは一つ——

> **「このデザイナーは、美しさと結果を同時にデザインできる。」**

---

### テーマ：Handcrafted Clarity（手の温もり × 伝わる明快さ）

| 要素 | 方向性 |
|---|---|
| **全体の質感** | 紙の温もりを感じるオフホワイトベース。完全なフラットデザインではなく、微細なノイズテクスチャ（Opacity 2〜4%）をオーバーレイ |
| **レイアウト** | 完全なグリッド整列を意図的に崩す脱グリッド・アシンメトリー。カード要素に -1°〜1° の微妙な回転を加え、「手で並べた」感覚を演出 |
| **形状** | 真円や完全な長方形を避け、`border-radius: 50% 50% 50% 50% / 60% 40% 60% 40%` のような非対称なオーガニックシェイプを採用 |
| **動き** | 等速移動（linear）は一切使用しない。物理世界の摩擦を感じさせる `cubic-bezier(0.25, 1, 0.5, 1)` をベースとし、要素が「ふわっと着地する」ようなイージングを全体に適用 |

---

### キーカラー

```
Background (ベース):  #F9F6F0  Warm Off-White   ──── 親近感・温かみを演出する主役
Primary Text (本文):  #2C2C2C  Ink Black        ──── コントラスト比4.5:1をクリアしつつ、純黒の圧迫感を回避
Accent (強調・CTA):   #E07A5F  Terracotta       ──── 感情を穏やかに刺激し、視線をCTAへ誘導
Sub Accent (補助):    #81B29A  Sage Green       ──── 成果・信頼を表現するセカンドアクセント
```

> **コントラスト設計原則:** テキストと背景のコントラスト比は最低 **4.5:1（WCAG AA基準）** を厳守し、屋外・視覚障害があるユーザーにも情報が届く設計を徹底する。

---

### タイポグラフィ

| 用途 | フォント | 設定値 |
|---|---|---|
| **見出し（Heading）** | Shippori Mincho / 游明朝 | `letter-spacing: 0.04em` / `font-weight: 500` |
| **本文（Body）** | Noto Sans JP | `line-height: 1.8` / `letter-spacing: 0.02em` |
| **欧文アクセント** | Cormorant Garamond | 見出しの英語サブテキストに使用 |

> 明朝体の見出しは「職人の手仕事」を想起させ、温かみと格調を同時に与えます。本文はゴシックで視認性を確保し、段落間には十分なホワイトスペースを設けてゲシュタルトの近接原則を適用します。

---

### マイクロインタラクションのアイデア

- **スクロール演出:** セクションが視界に入る直前に、要素が下から「ふわっと」浮き上がるフェードイン（Y軸 +20px → 0px、opacity 0→1）
- **CTAホバー:** ボタンは `transform: translateY(-4px)` でわずかに持ち上がり、「触れた」感覚を提供（Fittsの法則に従い、クリック領域は削らない）
- **作品カードホバー:** 微妙に `rotate(-0.5deg)` が解除されて正位置に戻り、「手に取った」ような感覚
- **カーソル:** テキストエリアではオリジナルカーソルを表示し、ブランドアイデンティティを演出

## 2. サイトの全体構成（サイトマップ）

### 構成判断：ワンページ構成 ＋ 作品詳細のみ別ページ

**理由:**
マーケティング責任者や採用担当者は多忙であり、複数のページを巡回する時間的余裕がありません。メインコンテンツはすべてワンページにまとめて流し読みを可能にしつつ、**作品詳細のみ個別ページ**として深掘りできる構造が最も効果的です。

---

### セクション構成と順序

```
[ TOP ] ────────────────────────────────────────────────────
  │
  ├─ 00. Navigation (固定ヘッダー)
  │      ロゴ | Works | About | Process | Contact
  │
  ├─ 01. Hero（ファーストビュー）
  │      キャッチコピー / サブコピー / CTA（仕事の依頼はこちら）
  │      ※訪問者の98%はここで「読むか/離脱するか」を決める最重要エリア
  │
  ├─ 02. Selected Works（厳選実績）
  │      3〜5件の代表作をカードグリッドで展示
  │      各カードクリックで作品詳細ページへ遷移
  │
  ├─ 03. Numbers（数字で語る実績）
  │      完了案件数 / 平均CVR改善率 / クライアント満足度など
  │      ※「美しさ+結果」を証明するセクション
  │
  ├─ 04. About（自己紹介）
  │      ブランドストーリー / 理念 / 得意領域
  │      写真：カメラ目線ではなく、作業中のCandid shot
  │
  ├─ 05. Process（仕事の進め方）
  │      ヒアリング → 設計 → デザイン → 納品 の4ステップ
  │      ※「一緒に仕事をした時のイメージ」をクライアントに持たせる
  │
  ├─ 06. Testimonials（クライアントの声）
  │      2〜3件の実際の声。顔写真あり / 会社名あり が理想
  │
  ├─ 07. Contact（問い合わせ）
  │      シンプルなフォーム + SNSリンク
  │      ※Fittsの法則：フォームとボタンは近接配置、タップ領域44px以上
  │
[ BOTTOM ] ─────────────────────────────────────────────────

[ 別ページ：作品詳細 /works/[slug] ]
  各LP案件の詳細ページ（次章で詳述）
```

## 3. 「作品（実績）詳細」の魅せ方

### 基本方針：「画像集」ではなく「プロジェクトの物語」を語る

単なるスクリーンショットの羅列は、駆け出しデザイナーと差別化できません。**「なぜこのデザインにしたのか」という思考プロセス**を語ることで、はじめて「デザインの結果を出せる設計力」が証明されます。

---

### 作品詳細ページ テンプレート（ワイヤーフレーム構成）

```
────────────────────────────────────────────────────────────
01. PROJECT HEADER（プロジェクトタイトルエリア）
────────────────────────────────────────────────────────────
  [ クライアント業種 ] [ 制作年月 ] [ 担当領域（例：UX設計/デザイン）]

  # プロジェクト名 / LP名称

  > ひとこと概要（20〜30字。例：「CV率を2倍にした化粧品ブランドのLPリデザイン」）

────────────────────────────────────────────────────────────
02. HERO IMAGE（ファーストビジュアル）
────────────────────────────────────────────────────────────
  LPのファーストビュー（Above the fold）部分の大きなモックアップ画像
  ※PCとスマホの2画面表示が理想

────────────────────────────────────────────────────────────
03. PROJECT BRIEF（プロジェクト概要）
────────────────────────────────────────────────────────────
  ┌────────────────┬──────────────────────────────────────┐
  │  課題 / 背景    │ 依頼前の状態。何が問題で、何を改善したかったか │
  ├────────────────┼──────────────────────────────────────┤
  │  ターゲット     │ 誰に届けるLPか。ペルソナの概要          │
  ├────────────────┼──────────────────────────────────────┤
  │  私の役割       │ ディレクション / UX設計 / デザイン など  │
  └────────────────┴──────────────────────────────────────┘

────────────────────────────────────────────────────────────
04. DESIGN STRATEGY（デザイン戦略・意図）
────────────────────────────────────────────────────────────
  ● 課題を解決するために「なぜこのデザインにしたか」の思考プロセス
  ● 配色・フォント・レイアウトの選定理由
  ● 特に工夫したUX上のポイント（例：CVRを高めるためのCTA設計）

────────────────────────────────────────────────────────────
05. FULL PAGE SCROLL（LP全体スクロール画像）
────────────────────────────────────────────────────────────
  LP全体を縦長で表示。訪問者が「デザインの全体観」を把握できる

────────────────────────────────────────────────────────────
06. DETAIL CLOSE-UP（こだわりセクションの拡大展示）
────────────────────────────────────────────────────────────
  2〜3箇所のこだわりポイントを拡大画像＋解説テキストで展示
  例：
  ・「Heroセクション：視線をCTAへ誘導するためのZ字レイアウト」
  ・「フォームUI：入力ハードルを下げる3ステップ分割設計」
  ・「Social Proof：不安解消のためのアイコン・数値の配置意図」

────────────────────────────────────────────────────────────
07. RESULTS（成果・数値）
────────────────────────────────────────────────────────────
  ★ CV率：リリース前 X% → リリース後 X%（+XX%改善）
  ★ クライアントのコメント（一言）

  ※数値が出せない場合は「クライアントから高評価を頂きました」では
  なく、「デザイン施策の意図と仮説」を代わりに丁寧に語る

────────────────────────────────────────────────────────────
08. NEXT PROJECT（次の作品へ誘導）
────────────────────────────────────────────────────────────
  ← 前の作品へ     次の作品へ →
```

## 4. 魅力的なコピーライティング（Hero セクション）

### 基本方針
ターゲット（マーケティング責任者・採用担当者）は **「美しさ+結果」** を求めています。「きれいなデザインが作れます」ではなく、**「あなたのビジネスに変化をもたらせます」** というメッセージが刺さります。

---

### パターン A ｜「結果にフォーカス」型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  デザインで、
  伝わる力に変える。

サブコピー（H2 / リード文）:

  Sunny Side Designは、見た目の美しさだけでなく
  「なぜ人はここで動くのか」を設計するLP専門スタジオです。
  心理学と感性を掛け合わせた戦略的デザインで、
  あなたのサービスに本質的な価値を与えます。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**強み:** 「結果を出す」ことへの期待を明確に喚起。「なぜ人は動くのか」というフレーズで専門性をアピール。

---

### パターン B ｜「感情・温もりにフォーカス」型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  完璧な画面より、
  温かい体験を。

サブコピー（H2 / リード文）:

  AIが生み出す「完璧なデザイン」に、人は慣れました。
  Sunny Side Designは、あえて揺らぎのある手仕事を選び、
  ユーザーの心に「ここは信頼できる」と感じさせるLPを作ります。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**強み:** AI時代の文脈を逆手に取り、独自の差別化ポジションを明確化。「Crafted, not Computed」の哲学を伝える。

---

### パターン C ｜「問いかけ・共感」型

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
キャッチコピー（H1）:

  そのLPで、
  本当に届いていますか？

サブコピー（H2 / リード文）:

  デザインはアートではなく、コミュニケーションです。
  Sunny Side Designは、ターゲットの心理を読み解き、
  「見て、信じて、動く」導線をひとつひとつ丁寧に設計します。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**強み:** 訪問者（マーケ責任者）が今まさに抱えている悩みに直接語りかける。問いかけ型で思わず続きを読ませる構造。

---

### 推奨パターン

> **パターン C → パターン A の順番で、A/Bテストを実施することを推奨します。**
> 認知（気づき）を促すパターンCと、行動（依頼）を促すパターンAは、ターゲットの文脈によって効果が変わります。まずはパターンCで「共感のフック」を作り、リニューアル時にパターンAで「成果訴求」に切り替えるロードマップが最も堅実です。

## 付録：実装優先度マップ

| 優先度 | セクション | 理由 |
|---|---|---|
| ★★★ | Hero + Selected Works | 判断の90%はここで決まる |
| ★★★ | 作品詳細ページ（2〜3件） | 質の高いクライアントほど深く読む |
| ★★☆ | Numbers + Testimonials | 信頼の証拠として必須 |
| ★★☆ | Contact | CVポイント。シンプルに、近く、大きく |
| ★☆☆ | About + Process | ファンになった人が読む。後回し可 |
